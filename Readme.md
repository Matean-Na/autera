# Domain Modules Overview

Проект построен на **модульной доменной архитектуре**, где каждый модуль отвечает
за одну бизнес-задачу и не пересекается по ответственности с другими.

Ключевые доменные модули:

- `ads` — объявления автомобилей (витрина)
- `inspections` — процесс проверки автомобиля
- `reports` — итоговый отчёт и доверительный результат

---

## 1️⃣ ads — объявления автомобилей

### Смысл

`ads` — это **витрина**.  
Всё, что связано с автомобилем **как товаром**.

Если убрать проверки, отчёты и роли —  
`ads` уже может существовать как простой маркетплейс.

---

### Что хранит ads

- марка / модель
- год выпуска
- пробег
- цена
- VIN
- город
- фотографии
- статус объявления
- статус проверки

```text
Ad
 ├─ brand / model
 ├─ year
 ├─ mileage
 ├─ price
 ├─ vin
 ├─ city
 ├─ photos
 ├─ status            (draft / moderation / published / rejected)
 └─ inspection_status (none / requested / done / certified)
```

## 2️⃣ inspections — процесс проверки автомобиля

### Смысл

`inspections` — это **процесс**, а не отчёт.  
Рабочий процесс осмотра автомобиля.

Если `ads` — это **товар**,  
то `inspections` — это **экспертиза товара**.

---

### Что хранит inspections

- к какому объявлению относится проверка
- кто продавец
- кто оценщик
- текущий статус проверки

```text
Inspection
 ├─ ad_id
 ├─ seller_id
 ├─ inspector_id
 └─ status (requested / assigned / in_progress / submitted)
```

## 3️⃣ reports — результат проверки (доверительный отчёт)

### Смысл

`reports` — это **результат** и **артефакт доверия**.

То, что:

- видит покупатель
- можно открыть через ссылку
- можно сохранить в PDF

---

### Что хранит reports

- итоговую оценку
- категорию состояния  
  *(плохое / хорошее / отличное)*
- привязку к проверке

```text
Report
 ├─ inspection_id
 ├─ total_score
 └─ label ("плохое" / "хорошее" / "отличное")
